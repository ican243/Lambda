# KOSPI·KOSDAQ 투자전략 분석 및 모의 자동매매 시스템

## 1. 프로젝트 소개

본 프로젝트는 **KOSPI·KOSDAQ 종목의 과거 주가를 이용해 투자전략을 검증하고**, 현재 주가를 실시간으로 모니터링하여 전략 조건이 충족되면 **한국투자증권 KIS 모의투자 API로 주문을 실행하는 웹 플랫폼**입니다.

### 한 줄 소개

> 과거 주가로 투자전략을 백테스트하고, 실시간 시세를 감시하여 조건에 맞는 순간 모의 매수·매도를 실행하는 시스템입니다.

---

## 2. 프로젝트 목표

- 국내 주식의 과거 주가 데이터 수집
- 이동평균선과 RSI를 이용한 투자전략 구현
- 전략별 수익률과 위험성 검증
- 실시간 주가 모니터링
- 전략 조건 충족 시 모의투자 주문 실행
- 주문 및 백테스트 결과 저장·조회
- 향후 실전 자동매매 시스템으로 확장 가능한 구조 설

---

## 3. 백테스트란?

백테스트는 투자전략을 **과거 주가 데이터에 적용해 실제로 수익이 났을지 검증하는 과정**입니다.

예를 들어 다음과 같은 전략을 정할 수 있습니다.

```text
5일 이동평균선이 20일 이동평균선을 위로 돌파하면 매수
5일 이동평균선이 20일 이동평균선을 아래로 돌파하면 매도
```

시스템은 과거 날짜를 순서대로 확인하며 가상의 매수와 매도를 실행하고, 다음 결과를 계산합니다.

- 총수익률
- 최종 자산
- 매매 횟수
- 승률
- 최대 낙폭(MDD)
- Sharpe Ratio
- 벤치마크 대비 성과

> 백테스트 결과가 좋다고 실제 투자에서도 반드시 수익이 나는 것은 아닙니다. 과거 데이터를 이용한 검증 결과이며, 실제 시장에서는 수수료, 세금, 주문 지연과 시장 상황 변화가 발생할 수 있습니다.

## 4. 구현한 투자전략

### 5.1 MA20 전략

`MA20`은 최근 20일 종가의 평균인 **20일 이동평균선**입니다.

```text
현재 주가가 MA20 위로 올라가면 → 상승 흐름으로 판단
현재 주가가 MA20 아래로 내려가면 → 하락 흐름으로 판단
```

단순한 구조라 이해하기 쉽지만, 주가가 이동평균선 주변을 반복해서 움직이면 매수·매도 신호가 자주 발생할 수 있습니다.

---

### 5.2 골든크로스 전략

단기 이동평균선이 장기 이동평균선을 위로 돌파하는 순간을 골든크로스라고 합니다.

```text
단기 이동평균선이 장기 이동평균선을 상향 돌파 → 매수 신호
단기 이동평균선이 장기 이동평균선을 하향 돌파 → 매도 신호
```

주가 상승 흐름을 확인하는 대표적인 추세 전략이지만, 신호가 늦게 발생할 수 있습니다.

---

### 5.3 RSI 전략

RSI는 최근 주가가 너무 많이 올랐는지 또는 너무 많이 떨어졌는지를 `0~100` 사이 숫자로 나타내는 지표입니다.

```text
RSI 70 이상 → 과매수 구간으로 판단 → 매도 고려
RSI 30 이하 → 과매도 구간으로 판단 → 매수 고려
```

RSI 값만으로 주가의 상승과 하락을 확정할 수는 없기 때문에 다른 지표와 함께 사용할 수 있습니다.

---

### 5.4 MA20 + RSI 필터링 전략

MA20에서 발생한 신호를 RSI로 한 번 더 확인하는 결합 전략입니다.

```text
주가가 MA20 위로 올라감
그리고 RSI 조건도 충족
→ 매수 신호

주가가 MA20 아래로 내려감
그리고 RSI 조건도 충족
→ 매도 신호
```

두 조건을 함께 사용해 불필요한 신호를 줄이는 것이 목적입니다.

## 6. 실시간 주가와 자동매매 방식

실시간 주가는 한국투자증권의 **KIS WebSocket API**를 이용해 수신합니다.

WebSocket은 프로그래밍 언어가 아니라 서버와 연결을 유지하며 데이터를 실시간으로 주고받는 통신 방식입니다.

```text
KIS WebSocket
    ↓
실시간 체결가 수신
    ↓
현재 전략 조건 검사
    ↓
매수·매도 신호 생성
    ↓
KIS REST 주문 API 호출
```

- **WebSocket API**: 실시간 주가 수신
- **REST API**: 매수·매도 주문 전송
- **KIS**: Korea Investment & Securities의 API 서비스

거래가 발생할 때마다 가격 데이터가 들어오므로 5초 또는 10초마다 다시 요청하는 방식과 다릅니다.

## 12. 한계점

- 과거 수익률이 미래 수익을 보장하지 않음
- 전략이 특정 과거 데이터에 과도하게 맞춰질 수 있음
- 실제 거래에서는 세금, 수수료, 슬리피지와 주문 지연이 발생
- API 장애나 WebSocket 연결 종료에 대한 대응 필요
- 자동매매 전 손실 한도와 주문 제한 등 리스크 관리 필수
- 현재는 모의투자 기반이며 실전 자동매매는 향후 검토 단계
